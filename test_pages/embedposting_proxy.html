Privly Embed-posting Proxy Test File

This file contains a test cases for different circumstances in which the Privly embed-posting dialog should correctly show submiting button and forward ENTER keyboard events.

Test cases are intended to be tested automatically.
To test it manually, please click the Privly posting button in each editable element to pop up the embed-posting dialog. Detailed expectations are listed in each test case.

1. Facebook status

You should see submit button in embed-posting dialog (caption: 发布).
You should see submitted when: Command+ENTER (Mac OS), Control+ENTER (Other OS).


2. Facebook chat

You should NOT see submit button in embed-posting dialog.
You should see Done button in embed-posting dialog.
You should NOT see submitted when: Shift+ENTER.
You should see submitted when: (Any other)+ENTER.


3. GitHub

GitHub only listens keydown event when the textarea is focused. This test case simulate this circumstance.

You should see submit button in embed-posting dialog (caption: Comment on this commit).
You should see submitted when: Command+ENTER or Control+ENTER.


4. Google Groups / Google Mail

You should NOT see submit button in embed-posting dialog.
You should see Done button in embed-posting dialog.

Post

5. Google Plus

You should NOT see submit button in embed-posting dialog.
You should see Done button in embed-posting dialog.

Share

6. Twitter

You should NOT see submit button in embed-posting dialog.
You should see Done button in embed-posting dialog.
You should see submitted when: Command+ENTER or Control+ENTER.
